## TCP/IP 프로토콜

### TCI/IP 프로토콜이란?

🔹 Transmission Control Protocol/Internet Protocol<br>
🔹 이 기종 간 네트워크 환경에 대한 표준이다.<br>
🔹 미국 **ARPANET**에서 개발된 프로토콜이며, OSI보다 먼저 만들어지고 가장 많이 사용되고 있다.<br>
🔹 인터넷에서 사용되고 있으며 다양한 네트워크와 상호 작용이 가능하다.<br>

> **ARPANET이란?**<br> 미국 국방성에서 국방 관련 기관 간에 정보 공유를 위해 추진한 프로젝트로, 원격 로그인, 파일 전송, 전자우편 등의 기능을 지원하는 네트워크이다.

| ---------------------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |

| Transport                          | Transport      | 🔹 도착하고자 하는 시스템까지 데이터를 전송한다.<br>🔹 프로세스를 연결해서 통신한다.<br>🔹 TCP, UDP                                                          |
| Network                            | Internet       | 🔹 논리적 주소인 IP주소를 부여해 Datagram을 정의하고 최단 경로를 결정한다.<br>🔹 IP주소를 읽어 경로를 결정하는 Routing을 담당한다.<br>🔹 IP, ARP, RARP, ICMP |

### TCP/IP 프로토콜 스니핑

🔹 **스니핑(Sniffing)** 도구를 사용해 웹사이트로 송수신되는 패킷을 모니터링 해보면 다음과 같은 과정을 볼 수 있다.<br>

1. 웹 사이트이므로 애플리케이션 계층에서 HTTP 프로토콜을 사용한다.
2. HTTP는 내부적으로 TCP 프로토콜을 사용하므로 전송 계층에서는 TCP 프로토콜이 나타난다.
3. 인터넷 계층에서 IP주소를 부여한다.
4. 네트워크 접근 계층에서 이더넷(Ethernet)으로 메시지를 전송한다.

> **스니핑(Sniffing)이란?**<br>네트워크에서 전송되는 데이터 패킷을 가로채고 분석하는 행위를 말한다.<br> 주로 패킷 감청 혹은 트래픽 분석을 목적으로 수행되며, 정보 탈취에 악용되기도 한다.

## 네트워크 접근 계층(Network Access)

### 네트워크 접근 계층이란?

🔹 LAN카드의 물리적 주소인 MAC주소를 가지고 전기적인 비트 신호로 메시지를 전송하는 계층이다.<br>
🔹 프레임(Frame)을 비트 단위로 전송한다.

### 프레임(Frame)의 구조

| ----------------------------- | ----------------------------------------------------------------------------------------- |
| Preamble                      | 동기화 정보를 가지고 있다.                                                                |
| SOF(Starting Frame Delimiter) | 프레임의 시작을 알리는 구분자이다.                                                        |
| Destination Address           | 수신자의 MAC주소이다.                                                                     |
| Source Address                | 송신자의 MAC주소이다.                                                                     |

| FCS(Frame Check Sequence)     | 비트열의 오류를 검사하기 위해 사용한다.                                                   |

## 인터넷 계층(Internet)

### 인터넷 계층이란?

🔹 송신자의 IP 주소와 수신자의 IP 주소를 읽어 경로를 결정하거나 전송하는 역할을 수행한다.<br>

#### 인터넷 계층의 기능

🔹 **Routing** : 수신자의 IP 주소를 읽어서 목적지까지 가는 것에 대한 최적의 경로를 판단한다.<br>
🔹 Point to Point Packet 전달<br>
🔹 논리 주소 지정(Logical Addressing) : IP 주소로 변환해서 사용한다.<br>
🔹 주소 변환(Address Transformation) : URL을 IP 주소로 변환해서 사용한다.<br>
🔹 기존 패킷에 IP Header를 붙여 데이터그램을 생성한다.<br>

#### 데이터그램(Datagram)의 구조

| Message | TCP 혹은 UDP | IP Header |
| ------- | ------------ | --------- |

### IP(Internet Protocol)

#### IP 프로토콜이란?

🔹 TCP/IP망의 네트워크 계층의 주소화, 데이터그램 포맷, 패킷 핸들링 등을 정의해놓은 인터넷 규약이다.<br>
🔹 IPv4와 IPv6가 사용 중이다.<br>

> **IPv4 vs IPv6**<br>
> |비교 |IPv4|IPv6|
> |---|---|---|
> |주소 길이 |32비트 |128비트|
> |주소 체계|10진수 (255.255.255.255)|16진수 (1234:1234:1234:1234:0000:0000:ffff:1111)|
> |지원하는 통신 방식|멀티캐스트, 유니캐스트, **브로드캐스트** |멀티캐스트, 유니캐스트, **애니캐스트**|

#### IP Header 구조

🔹 **Version** : IPv4, IPv6를 구분하는 정보이다.<br>
🔹 **Header LENgth** : 헤더의 전체 길이를 의미한다.<br>
🔹 **Type of Service** : 서비스 유형을 의미한다.<br>
🔹 **Identification** : 송신자에서 보낸 Datagram을 식별하기 위한 정보이다.<br>
🔹 **Total Length** : IP Datagram의 Byte 수이다.<br>
🔹 **Flag & Fragment Offset** : 패킷의 크기가 클 경우 분할되는데, 분할 될 경우 수신자가 재조합해야 하기 때문에 알려주기 위한 정보이다.<br>
🔹 **TTL(Time to Live)** : IP 패킷이 통과할 수 있는 라우터의 수를 의미한다. 라우터 하나를 통과할 때 마다 1씩 감소하며, 0이 되면 패킷은 자동 폐기된다.<br>
🔹 **Protocol** : IP Header 위의 상위 프로토콜 종류를 알린다. ICMP or TCP or UDP.<br>
🔹 **Header Checksum** : 헤더의 무결성 검사를 위한 정보이다.<br>

#### IP 단편화(Fragmentation)

🔹 **MTU(Maximum Transmission Unit)** : 한 번에 통과할 수 있는 패킷의 최대 크기를 의미한다.<br>
🔹 MTU 값보다 패킷의 크기가 크면 패킷이 분할되며, 그 정보를 Flag & Offset에서 가지고 있게 된다.<br>

### IP 주소와 서브넷 마스크(Subnet Mask)

#### 서브넷 마스크(Subnet Mask)

🔹 IP 주소는 클래스(Class)로 분류된다.<br>
🔹 클래스는 네트워크 ID와 호스트 ID로 분류할 수 있다.<br>
&emsp; 🔹 **네트워크 ID** : 네트워크 주소 부분이다..<br>
&emsp; 🔹 **호스트 ID** : 하나의 네트워크에 부여될 수 있는 호스트 IP 주소의 자릿수이다.<br>

#### 클래스(Class) 구조

_그림 추가 필요_<br>
🔹 클래스는 IPv4 주소를 네트워크 크기에 따라 분류하는 체계이다.
|클래스|네트워크 ID 길이 | 호스트 ID 길이| 사용 범위|
|---|---|---|---|
|Class A| 8비트 | 24비트| 대규모 네트워크(국가, 대기업)|
|Class B| 16비트| 16비트 | 중간 규모 네트워크(대학, 중견기업)|
|Class C| 24비트 | 8비트 | 소규모 네트워크(가정, 소기업)|
|Class D| **멀티캐스트** 전용| 사용 불가 | 멀티캐스트 |
|Class E| 실험용 | 사용 불가 | 연구 목적|

> **멀티캐스트란?**<br>특정 그룹에 속한 여러 개의 장치에게 동시에 데이터를 전송하는 방식이다. (1:N)

#### 서브넷팅(Subneting)

🔹 주어진 네트워크 주소를 작게 나누어 여러 개의 서브넷으로 구성하는 과정이다.<br>
🔹 서브넷팅을 활용하면 필요한 만큼의 주소를 나눠서 사용할 수 있으므로 IP 주소의 낭비를 줄여 효율을 높일 수 있으며, 네트워크 관리 및 보안의 용이성을 챙길 수 있다.<br>
🔹 **서브넷 마스크(Subnet Mask)** : 네트워크 식별자 부분을 구분하기 위한 Mask이다.<br>

_예제 문제 풀고 과정 작성 예정_

> **CIDR(Classless Inter-Domain Routing)표기법이란?**<br>
> IP 주소 뒤에 `/n`을 붙여 네트워크 주소와 호스트 주소를 구분하는 표기법이다.<br>
> | CIDR 표기법 | 서브넷 마스크 | 네트워크 비트 | 호스트 비트 | 최대 호스트 수 |
> |------------|-----------------|-------------|-------------|--------------|
> | /8 | 255.0.0.0 | 8비트 | 24비트 | 16,777,214 |
> | /16 | 255.255.0.0 | 16비트 | 16비트 | 65,534 |
> | /24 | 255.255.255.0 | 24비트 | 8비트 | 254 |
> | /30 | 255.255.255.252 | 30비트 | 2비트 | 2 |

### 라우팅(Routing)

#### 라우팅이란?

🔹 네트워크 간 패킷을 최적 경로로 전달하는 과정이다.<br>
🔹 목적지에 대한 경로 정보(Routing Table)을 참고하여 목적지까지 가는 최적의 경로를 결정한다.<br>
🔹 결정된 경로를 기반으로 **포워딩(Forwarding)** 을 수행하게 된다.<br>

> **포워딩(Forwarding)이란?**<br>
> 결정된 경로를 기반으로 패킷을 실제로 전송하는 과정을 말한다.

> **라우팅 테이블 확인하기**<br>
> 윈도우 시스템에서 `netstat -r`로 확인할 수 있다.<br>
> 이외에도 `route PRINT`명령어로도 확인할 수 있다.

#### 정적 라우팅과 동적 라우팅

| 구분                              | 설명                                                                                                                                                                                                                                                                              |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 정적 경로 라우팅(Static Routing)  | 🔹 관리자가 수동으로 경로를 설정하는 방식이다.<br>🔹 경로 설정이 실시간으로 이루어지지 않는다.<br>🔹 관리자가 최적의 경로 설정을 해 라우터의 직접적인 처리 부하를 감소시키기 때문에 소규모 네트워크에 적합하다.<br>🔹 다만 관리 부담이 크다.                                      |
| 동적 경로 라우팅(Dynamic Routing) | 🔹 관리자의 개입 없이 네트워크의 상황 변화에 따라 인접 라우터 간 자동으로 경로 정보를 교환한다.<br>🔹 라우팅 알고리즘을 통해 경로 설정이 실시간으로 이루어진다.<br>🔹 유지보수가 쉬우나 리소스 사용량이 많다.<br>🔹 Distance Vector Routing, Link State Routing 등의 기법이 있다. |

#### IGP와 EGP 라우팅 프로토콜

🔹 라우팅 프로토콜이란 경로를 결정하는 알고리즘을 포함한 프로토콜이다.<br>
|구분|설명|
|---|---|
|IGP(Intertal Gateway routing Protocol)|🔹 한 도메인 내에서 라우팅 정보를 교환해 경로를 결정하는 프로토콜이다.|
|EGP(Exterior Gateway routing Protocol)|🔹 도메인 간에 라우팅 정보를 교환해 경로를 결정하는 프로토콜이다.|

### 라우팅 프로토콜의 종류

#### Distance Vector와 Link State

| 구분           | Distance Vector                                                                                                 | Link State                                                                                                                                                       |
| -------------- | --------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 설명           | 통과해야 하는 라우터의 수가 적은 쪽(최단 거리)으로 경로를 정하는 방법이다.                                      | 네트워크 대역폭, 지연 정보 등을 종합적으로 고려해 Cost를 산정하고, 해당 Link의 Cost에 따라 경로를 정하는 방법이다.                                               |
| 알고리즘       | 최단 경로를 구하는 벨만 포드(Bellman-Ford)알고리즘 기반                                                         | 최소 신장트리를 구하는 다익스트라(Dijkstra) 알고리즘 기반                                                                                                        |
| 동작 원리      | 네트워크 변화 발생 시 해당 정보를 인접 라우터에 정기적으로 전달하고, 인접 라우터는 라우팅 테이블을 갱신한다.    | 네트워크 변화 발생 시 링크 상태 변경 정보를 인접 라우터에게 즉각 전달하고 이를 저장한다.                                                                         |
| 라우팅 정보    | 모든 라우터까지의 거리 정보                                                                                     | 인접 라우터까지의 Link Cost 계산                                                                                                                                 |
| 정보 전송 시점 | 일정 주기(30초, 인접 라우터와 공유)                                                                             | 변화 발생 시에만 전송                                                                                                                                            |
| 대표 프로토콜  | **RIP**, IGRP, EIGRP(내부 라우팅), **BGP**                                                                      | **OSPF**, IS-IS                                                                                                                                                  |
| 단점           | 🔹 모든 변화 정보를 주기적으로 갱신하므로 망 자체 트래픽을 유발한다.<br>🔹 **Routing Loop** 발생 가능성이 있음. | 🔹 네트워크 전체 정보 유지를 위해 메모리가 많이 소요된다.<br>🔹 라우터의 CPU 사용량이 높아진다.<br>🔹 라우터 간 회선의 대역차이로 동기화를 실패할 가능성이 있다. |

> **BGP(Border Gateway Protocol)란?**<br>
> 자율 시스템 간의 라우팅 프로토콜이다.<br>
> 주로 인터넷에서 많이 사용되는 외부 라우팅 프로토콜로 서로 다른 자율 시스템에서 동작하는 라우팅 프로토콜이다.<br>

> **Routing Loop란?**<br>
> 패킷이 목적지에 도달하지 못하고 네트워크 내에서 무한히 반복해서 전달되는 현상이다.<br>
> 주로 잘못된 라우팅 정보나 라우팅 프로토콜의 오류로 인해 발생한다.

### RIP(Routing Information Protocol)

#### RIP란?

🔹 Distance Vector(거리 벡터)기반의 라우팅 프로토콜이다.<br>
🔹 통과해야 하는 라우터 수를 **홉 수(Hop Count)** 라 하고, 홉 수가 16Hop을 넘으면 패킷을 폐기한다.<br>
🔹 180초 이내에 새로운 라우팅 정보가 수신되지 않으면 해당 경로를 이상 상태로 간주한다.<br>
🔹 30초 단위로 RIP 브로드캐스팅을 통해 라우팅 테이블을 관리한다.<br>
🔹 라우팅 정보 변경 시 모든 망에 적용하므로 큰 규모의 망에는 적합하지 않다.<br>

### OSPF(Open Shortest Path First)

#### OSPF란?

🔹 Link State(링크 상태)기반의 라우팅 프로토콜이다.<br>
🔹 링크(Link)의 지연(Delay), 처리량(Throughput), 신뢰성(Reliability) 정보를 이용한다.<br>
🔹 네트워크를 Area로 구분하여 정보 교환 과다로 발생하는 라우터의 성능 저하를 예방하고 대역폭을 절약하기 때문에 대규모 IP망에서 사용된다.<br>
🔹 링크 변화 감지 시 변경된 링크에 대한 정보만을 모든 라우터에게 전달한다.<br>

#### OSPF 계위

🔹 OSPF는 **라우팅 계위** 를 만들어 라우팅을 수행하며, 라우팅 계위는 OSPF에서 관리하는 라우터들 간에 그 역할을 정의한 것이다.<br>

_그림 추가 필요_<br>
🔹 **ABP(Area Border Router, 경계 영역 라우터)** : Area에 백본망을 연결하는 라우터이다.<br>
🔹 **ASBR(Autonomous System Boundary Router, 자율 시스템 경계 라우터)** : 다른 AS(Autonomous System)에 속한 라우터와 경로 정보를 교환한다.<br>
🔹 **IR(Internal Router, 내부 라우터)** : Area에 접속한 라우터이다.<br>
🔹 **BR(Backbone Router, 백본 라우터)** : 백본망에 접속한 모든 라우터이다.<br>

### ICMP(Internet Control Message Protocol)

#### ICMP이란?

🔹 TCP/IP에서 오류를 제어하는 프로토콜이다.<br>
🔹 호스트 및 라우터는 다른 호스트나 라우터가 현재 도달 가능한지의 여부를 결정한다.<br>
🔹 라우터는 더 좋은 경로가 있음을 근원지 호스트에게 통지한다.<br>
🔹 호스트나 라우터는 처리가 불가능한 너무 빠른 데이터그램이 도착하면 이를 다른 시스템에게 통지한다.<br>

#### ICMP의 주요 기능

🔹 패킷 처리 중 발생한 오류를 보고한다.<br>
🔹 다른 호스트로부터 특정 정보를 획득하기 위해 사용한다.<br>
🔹 통신이 정상적인지 확인하고 두 호스트 간 에러 처리를 담당한다.<br>

#### ICMP 메시지 구조와 종류

_구조 그림 첨부 필요_<br>
🔹 **Type** : ICMP 메시지 유형을 표시한다.<br>
🔹 **Code** : TYpe과 함께 세부적인 유형을 표현한다.<br>
🔹 **Check Sum** : IP Datagram Checksum <br>

| Type     | Message                     | 설명                                                            |
| -------- | --------------------------- | --------------------------------------------------------------- |
| 3        | Destination Unreachable     | 라우터가 목적지를 찾지 못한 경우 보낸다.                        |
| 4        | Source Quench               | 패킷을 너무 빨리 보내 네트워크에 무리를 주는 호스트를 제지한다. |
| 5        | Redirection                 | 패킷 라우팅 경로를 수정한다.                                    |
| 8 or 10  | Echo Request or Reply       | 호스트의 존재를 확인한다.                                       |
| 11       | Time Exceeded               | 패킷을 보냈으나 시간이 경과해 삭제된 경우 보낸다.               |
| 12       | Parameter Problem           | IP 헤더에 잘못된 정보가 있음을 알린다.                          |
| 13 or 14 | Timestamp Request and Reply | Echo와 유사하나 시간에 대한 정보가 추가된 메시지이다.           |

#### TTL(Time To Live)의 역할

🔹 ICMP는 TTL을 사용한다.<br>
🔹 TTL 값은 라우터를 통과할 때 마다 1씩 감소하며, 0이 되면 패킷은 자동으로 폐기된다.<br>
🔹 패킷이 정해진 시간 내로 도착하지 않으면 ICMP는 시간 초과 메시지를 보낸다.<br>

### 데이터 전송 방식

| 구분      | 설명                                                                                                                         |
| --------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Unicast   | 1:1                                                                                                                          |
| Broadcast | 1:N<br>동일한 서브넷 상 모든 수신자에게 전송한다.                                                                            |
| Multicast | M:N<br>하나 이상의 송신자들이 특정 그룹의 수신자에게 전송한다.                                                               |
| Anycast   | 1:N 中 1 <br>그룹에 등록된 노드 중 최단 경로 노드 한 개에만 전송한다.<br>IPv6부터는 Broadcast가 없어지고 Anycast를 사용한다. |

### 멀티캐스트와 IGMP(Internet Group Management Protocol)

#### IGMP란?

🔹 멀티캐스트는 그룹에 등록된 사용자에게만 데이터를 전송한다<br>
🔹 IGMP는 이 그룹에 등록된 사용자를 관리하기 위한 프로토콜이다.<br>
🔹 멀티캐스트 시 멀티캐스트에 참석하는 수신자 정보를 제공한다.<br>
🔹 시작 호스트에서 수신을 받을 목적지 호스트들에게 메시지를 전송한다.<br>

#### IGMP 메시지 구조

_그림 추가 필요~_<br>
🔹 **Version** : IGMP프로토콜의 버전을 표시한다. (현재 버전은 2이다.)<br>
🔹 **Type** : 메시지 유형을 표시한다. (1 = 보고, 2 = 질의)<br>
🔹 **Group id** : 보고 메시지에서, 호스트에서 신규 가입하고자 하는 멀티캐스트 서비스의 Group id이다.<br>

### NAT(Network Address Translation)

#### NAT란?

🔹 사설 IP를 라우팅이 될 수 있는 공인 IP로 변경하는 주소 변환이다.<br>
🔹 내부 망에서는 사설 IP, 외부 망에서는 공인 IP를 사용해 공인 IP를 절약할 수 있다.<br>
🔹 내부를 사설 망으로 함으로써 보안성을 챙길 수 있다.<br>
🔹 ISP(Internet Service Provider) 변경에 따른 내부 IP 변경을 최소화 할 수 있다.<br>

#### NAT의 종류

🔹 NAT의 동작 방식 및 사용 목적으로 구분한 표
| NAT 종류 | 설명 | 특징 |
|----------|------|------|
| Normal NAT | N개의 사설 IP를 한 개의 공인 IP로 변환한다. | 대부분의 NAT (PAT 포함) |
| Reverse NAT (Static NAT) | 🔹 Normal NAT 설정만으로는 외부에서 내부 네트워크로 접속할 수 없다.<br>🔹 Reverse NAT를 통해 공인 IP를 사설 IP로 고정 매핑해준다. | 1:1 매핑, 외부에서 내부 서버 접근 가능하다. |
| Redirect NAT | 장애 발생 시 목적지 주소를 재지정할 경우 사용한다. | 특정 포트 트래픽을 내부 서버로 전달한다. |
| Exclude NAT | 특정 네트워크 트래픽을 NAT를 적용받지 않도록 한다. | NAT 없이 원본 IP를 유지한다. |

🔹 NAT의 변환 방식(주소 매핑 방식)으로 구분한 표<br>
_그림을 넣어야 할 것 같다_<br>
| NAT 종류 | 설명 | 특징 |
|----------|------|------|
| Dynamic NAT | 내부의 사설 미리 정해진 공인 IP로 랜덤 매핑하는 방법이다. | 사용 가능한 공인 IP 중 하나를 임시 할당한다.(= 모든 공인 IP가 사용중인 경우 나머지 사설 IP는 할당될 수 없다.) |
| Static NAT | 관리자가 직접 사설 IP와 공인 IP를 1:1로 고정 매핑하는 방법이다. | 특정 장치에 항상 동일한 공인 IP를 할당한다. |
| PAT (Port Address Translation) | 여러 개의 사설 IP를 포트 변환을 통해 하나의 공인 IP로 변환한다. (= 사설 IP들은 각각 포트를 다르게 사용한다.) | 가장 많이 사용되며, 공인 IP가 부족한 경우에도 사설 IP 매핑을 할 수 있다. |

### ARP(Address Resolution Protocol)

#### ARP란?

🔹 IP주소를 MAC주소로 변환하는 프로토콜이다.<br>
🔹 ARP Request와 ARP Reply를 통해 ARP Cache Table을 유지해 인접 컴퓨터의 IP 주소와 MAC 주소를 가진다.<br>

### ARP의 동작 방식과 Operation Code

1. ARP Request를 전체 네트워크에 브로드캐스트한다.
2. 대상 장치가 자신의 IP와 일치하는 경우 ARP Reply로 MAC주소를 회신한다.
3. MAC주소를 받은 호스트는 ARP Cache Table에 이를 저장해 다시 요청하지 않는다.

| Opcode | ARP Message Type |
| ------ | ---------------- |
| 1      | ARP Request      |
| 2      | ARP Reply        |
| 3      | RARP Request     |
| 4      | RARP Reply       |
| 5      | DRARP Request    |
| 6      | DRARP Reply      |
| 7      | InARP Request    |
| 8      | InARP Reply      |

> **ARP 명령어로 IP주소와 MAC주소 확인하기**<br>
> Linux : `arp` 명령어로 IP 주소, MAC 주소, Ethernet 인터페이스를 확인할 수 있다.<br>
> 윈도우 : `arp -a` 명령어로 IP주소, MAC 주소를 확인할 수 있다.<br>

### RARP(Reverse Address Resolution Protocol)

#### RARP란?

🔹 MAC주소로 IP주소를 알아내는 프로토콜이다.<br>
🔹 **Diskless Host**에서 주로 사용되었다.<br>

> **Diskless Host란?**<br>
> 자체 저장장치(SSD, HDD)가 없는 컴퓨터이다.<br>OS를 데이터나 로컬 저장소에서 부팅하는게 아닌, 네트워크를 통해 원격 서버에서 로드하여 실행하는 장치이다.<br>학교나 기업 환경, 클라우드 기반 가상 데스크톱 등에서 사용된다.

#### RARP의 동작 방식

1. 장비(클라이언트)는 자신의 MAC 주소만 알고 있고, IP 주소를 모르는 상태이다.
2. RARP Request를 네트워크에 브로드캐스트한다.
3. RARP Server가 요청을 받아 MAC 주소를 확인하고 해당 IP 주소를 RARP Reply로 회신한다.
4. 클라이어트는 회신받은 IP 주소를 사용해 네트워크 통신을 시작할 수 있게 된다.

> RARP Request는 브로드캐스트로 전송하며, RARP Reply는 유니캐스트로 전송한다.
